# Troubleshooting

## Common Issues

### "Failed to load config.yml"

**Symptom:** Admin page shows error about loading configuration.

**Check:**
1. `curl https://service-chine.pages.dev/admin/config.yml` returns valid YAML
2. YAML syntax is valid — use `yamllint` or `python -c "import yaml; yaml.safe_load(open('public/admin/config.yml'))"`
3. The `<link rel="cms-config-url">` in `admin.astro` has the correct `href`

### "Login with GitHub" doesn't appear

**Symptom:** Admin page loads but shows Netlify Identity login instead of GitHub.

**Check:** The `config.yml` backend section is correct:
```yaml
backend:
  name: github
  repo: sabrinabean/service-chine
  base_url: https://decap-proxy.<zone>
```

### GitHub OAuth redirect fails

**Symptom:** After authorizing on GitHub, redirected to an error page.

**Check:**
1. OAuth App callback URL is exactly `https://decap-proxy.<zone>/callback?provider=github`
2. The OAuth App Client ID and Secret are correct
3. Proxy Worker secrets are set (check Cloudflare Dashboard)
4. Proxy Worker is deployed and running

### "Repository not found" after login

**Symptom:** Authentication succeeds, but CMS shows "repository not found."

**Check:**
1. The OAuth App, repository, and GitHub user are all under the SAME account
2. `config.yml` `backend.repo` is correct
3. If repo is private, `GITHUB_REPO_PRIVATE` is set to `"1"` in `wrangler.jsonc`
4. OAuth App has the correct permissions (repo scope)

### Content not appearing after publish

**Symptom:** Published content doesn't show on the live site.

**Check:**
1. Commit was pushed to the `main` branch
2. GitHub Action ran successfully (check Actions tab)
3. Content file is at the correct path (`src/content/<collection>/`)
4. Frontmatter fields match `src/content.config.ts` Zod schema
5. `npm run build` succeeds locally with the new content

### Image upload fails

**Symptom:** Image widget shows an error when uploading.

**Check:**
1. The target directory exists (`src/assets/<collection>/`)
2. `media_folder` and `public_folder` are correctly configured in `config.yml`
3. The user has write access to the repository

### Build fails after migration

**Symptom:** `npm run build` fails with import errors.

**Check:**
1. `npm install` ran successfully after migration
2. Removed Keystatic imports from `astro.config.mjs`
3. Check if any code still imports from `@keystatic/*`

### dev server fails

**Symptom:** `npm run dev` doesn't start.

**Check:**
1. `decap-cms-app` is installed: `npm ls decap-cms-app`
2. Astro config is valid: `npx astro check`
3. No stale imports from removed packages

### decap-cms-app Vite compatibility warnings

**Symptom:** Console shows peer dependency warnings during `npm install`.

**Diagnosis:** These are normal. `decap-cms-app` bundles React 17-era widgets with peer dep ranges that don't include React 19. They work in practice — just minor console noise. Ignore unless the admin page fails to render.

### OAuth Proxy not deploying

**Symptom:** `deploy-proxy` job fails in GitHub Actions.

**Check:**
1. `proxy/package-lock.json` is committed (generated by `cd proxy && npm install`)
2. `CLOUDFLARE_API_TOKEN` has Workers permissions
3. `GITHUB_OAUTH_ID` and `GITHUB_OAUTH_SECRET` secrets are set in GitHub
4. `proxy/wrangler.jsonc` is valid JSONC (with comments)
